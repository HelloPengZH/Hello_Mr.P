提取
使用正则表达式可以进行匹配, 使用 exec 可以将匹配到的数据提取出来
语法:
正则表达式对象.exec( 字符串 ) -> 封装成数组

循环提取
在一个字符串中将所有复合的要求的字符串提取出来
1> 正则表达式需要使用全局模式
var r = new RegExp( '正则', 'g' );
var r = /正则/g;
2> 调用 exec 首先获得第一个匹配项
再调用一次该方法, 就可以获得 第二个匹配项
一直调用下去, 就可以得到所有的匹配项
直到最后全部匹配完, 如果还用该方法, 则返回 null


案例： 在 'abc123def456ghij789klum' 提取数字
var str ='abc123def456ghij789klum'
var r = /\d+/;
r.exec( str ) =>123
r.exec( str ) =>123
var r = /\d+/g;
r.exec( str ) => 123
r.exec( str ) => 456
r.exec( str ) => 789
r.exec( str ) => null
while ( res = r.exec( str ) ) {
// 操作 res, 处理捕获的结果
console.log( res )
}

元字符及部分用法


1 : -- . -- 表示任意 一个 非换行(即不包含回车) 符号
2 : -- () -- 表示分组和提高优先级
3 : -- [] -- 表示一个字符 即 匹配在"[ 字符 ]"内的任意一个字符 例: [ abc ] 表示一个字符 可以是 a , 或者 b ,或者 c 即 a || b || c

4 : -- | -- 或 表示 ...或者是...
5 : -- \ -- 转义字符 表示将 字符表示为字符本身的字面含含义
5 : -- * -- *之前的字符或字符组( abc )出现0次或多次 例 /123*/ 匹配的就是3的出现 可匹配 12中表示出现0次 123 中表示3出现1次 1233表示出现2次 12333表示出现3次 不匹配1 /m(111)*/ 表示匹配 m111 m111111 m 不匹配m11和 m1111
6 : --{}-- 表示前面的一个字符 出现了几次 /m{ 10 } / 出现10个m的进行匹配
7 : --(?:)-- 匹配并不添加到返回数组

8 : --\w-- 匹配所有的字符包括 数字 字符 下划线
8.I : --\W-- 不匹配字符 即所有的非数字 字符 下划线
9 : --\s-- 空白字符, 包括空格, tab, 回车换行等
10 : --\S-- 非空白字符
js 中常常使用 [\s\S] 表示任意字符
11 : --\D-- 非数字
12 : --\d-- 数字

13 : --^-- 以...开头 ^1s$ 以1开头 , 以s结尾
14 : --$-- 以...结尾
15 : --/(.*)\1/-- 反向引用 中"\1"表示第一个"("括号括起来的那一组 和之前完全匹配 即提取成对出项的结构
